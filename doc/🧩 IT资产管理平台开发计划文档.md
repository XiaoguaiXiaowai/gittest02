# 🧩 IT资产管理平台开发计划文档

## 一、项目背景与目标

### 1.1 项目背景

随着公司IT设备数量的不断增长，传统的Excel记录与人工管理方式已难以满足精细化管理需求。设备采购、分配、维修、报废等环节缺乏统一的追踪与统计机制，导致资产利用率低、审计困难、信息不一致等问题。

### 1.2 项目目标

开发一套基于Web的 **IT资产管理平台**，实现资产全生命周期管理，包括：

- **资产登记、分配、调拨、报废**等流程管理；

- **二维码资产标签打印与扫码识别**；

- **资产查询与可视化报表导出**；

- **权限控制与操作日志记录**；

- **提升资产管理效率与透明度**。

---

## 二、功能模块设计

| 模块名称         | 功能说明              | 主要功能点                 |
| ------------ | ----------------- | --------------------- |
| **用户与权限管理**  | 负责系统用户、角色与权限控制    | 用户登录注册、角色配置、权限分配、密码管理 |
| **资产登记与管理**  | 统一录入和管理公司所有IT资产信息 | 资产录入、修改、删除、导入导出、状态管理  |
| **资产查询与统计**  | 提供多维度查询与统计分析      | 条件筛选、模糊搜索、分页展示、资产分布图表 |
| **二维码打印与识别** | 通过二维码实现快速识别资产     | 自动生成资产二维码、批量打印、扫码绑定   |
| **资产报表导出**   | 生成Excel或PDF格式资产报表 | 报表模板选择、字段自定义导出        |
| **资产生命周期管理** | 跟踪资产从采购到报废的全过程    | 入库、分配、维修、退库、报废记录      |
| **日志与审计模块**  | 系统操作记录与安全审计       | 操作日志、变更追踪、导出记录        |

## 三、系统技术架构

### 3.1 整体架构

`+--------------------------+ |        前端层 (React)    | |  - Ant Design UI         | |  - Axios 请求             | +------------+-------------+              |              v +--------------------------+ |       后端层 (Django)    | |  - Django REST Framework | |  - JWT 用户认证          | |  - Celery 异步任务        | +------------+-------------+              |              v +--------------------------+ |      数据层 (MySQL)      | |  - 资产信息表            | |  - 用户与角色表          | |  - 日志与审计表          | +--------------------------+`

### 3.2 技术选型

| 层级    | 技术栈                       | 说明             |
| ----- | ------------------------- | -------------- |
| 前端    | React + Vite + Ant Design | 提供响应式UI与良好交互体验 |
| 后端    | Django + DRF              | 快速开发RESTful接口  |
| 数据库   | MySQL                     | 存储资产与用户数据      |
| 缓存与队列 | Redis + Celery            | 提升性能，支持异步任务    |
| 部署    | Docker + Nginx + Gunicorn | 实现可持续部署与负载均衡   |
| 版本管理  | Git + GitHub/GitLab       | 源码托管与版本控制      |
