# 🚀 IT资产管理平台迭代开发计划文档

## 一、总体说明

### 1.1 开发模式

项目采用 **敏捷迭代开发（Agile / Scrum）** 模式，每个迭代周期为 **2周**，共分为 **5个迭代阶段**。  
每次迭代包含以下环节：

- 迭代规划（Iteration Planning）

- 开发与联调（Development & Integration）

- 测试与验收（Testing & Review）

- 迭代回顾与优化（Retrospective）

### 1.2 项目周期概览

| 阶段  | 周期     | 主要目标        |
| --- | ------ | ----------- |
| 迭代1 | 第1-2周  | 环境搭建与基础框架开发 |
| 迭代2 | 第3-4周  | 用户管理与资产登记模块 |
| 迭代3 | 第5-6周  | 查询统计与二维码功能  |
| 迭代4 | 第7-8周  | 报表导出与日志审计模块 |
| 迭代5 | 第9-10周 | 测试优化与上线部署   |

---

## 二、迭代计划详情

### 🔹 **迭代1：项目启动与基础框架搭建**

**周期：** 第1-2周  
**目标：** 搭建项目开发与运行环境，完成系统基础架构与模板页面。

**主要内容：**

- 后端：Django项目初始化，配置MySQL数据库，定义基础模型结构（用户、资产表框架）。

- 前端：React项目创建，Ant Design组件库集成，完成登录页和主界面框架。

- DevOps：Git仓库初始化、Docker开发环境构建、CI/CD基础流水线配置。

**测试与交付：**

- 验证环境可用性；

- 完成基础页面访问与接口连通测试；

- 交付初版系统骨架。

---

### 🔹 **迭代2：用户与资产管理核心模块**

**周期：** 第3-4周  
**目标：** 实现用户登录、权限管理及资产登记的核心功能。

**主要内容：**

- 用户模块：注册、登录、JWT认证、角色权限管理；

- 资产模块：资产录入、编辑、删除、状态字段管理；

- 后端API设计与Swagger文档编制；

- 前后端接口联调。

**测试与交付：**

- 单元测试覆盖率≥70%；

- 用户登录及资产登记端到端测试；

- 发布 **v0.1 Beta** 内测版本。

---

### 🔹 **迭代3：资产查询与二维码打印功能**

**周期：** 第5-6周  
**目标：** 提供资产快速查询、统计展示与二维码功能。

**主要内容：**

- 查询模块：多条件查询、分页展示、模糊搜索；

- 二维码：自动生成资产二维码、批量打印、扫码识别；

- 统计模块：资产状态统计（按部门/类型/状态）；

- 性能优化与前端交互改进。

**测试与交付：**

- 集成测试与前端功能验收；

- 用户体验优化；

- 发布 **v0.2 Beta** 功能增强版。

---

### 🔹 **迭代4：报表导出与日志审计模块**

**周期：** 第7-8周  
**目标：** 增强数据可视化能力，完善系统安全与可追溯性。

**主要内容：**

- 报表模块：支持Excel、PDF导出，字段自定义选择；

- 日志模块：记录用户操作日志、导出日志；

- 权限审计功能实现；

- 数据库索引与查询性能优化。

**测试与交付：**

- 压力测试与性能验证；

- 安全性测试（认证与权限）；

- 发布 **v0.9 Release Candidate（RC）** 版本。

---

### 🔹 **迭代5：测试优化与部署上线**

**周期：** 第9-10周  
**目标：** 系统全面测试、Bug修复、上线部署。

**主要内容：**

- 全面系统测试（功能、性能、安全）；

- 修复问题、优化代码结构；

- 编写部署文档与用户手册；

- 部署到生产环境（Docker + Nginx + Gunicorn）。

**测试与交付：**

- 完成用户验收测试（UAT）；

- 正式发布 **v1.0 正式版**；

- 开始后续维护与版本升级计划。

---

## 三、测试与交付计划

| 测试类型        | 时间节点        | 主要内容            |
| ----------- | ----------- | --------------- |
| 单元测试        | 每个迭代周期内持续进行 | 后端接口单元测试、前端组件测试 |
| 集成测试        | 每次迭代结束时     | 模块间接口与数据流测试     |
| 用户验收测试（UAT） | 第10周        | 核心业务流程测试与用户体验评估 |
| 回归测试        | 每次迭代发布前     | 验证旧功能未受影响       |
| 性能与安全测试     | 第8-10周      | 并发性能、权限与安全漏洞检测  |

## 四、持续集成（CI）与持续交付（CD）流程概述

### 4.1 CI/CD 工具链

- **代码托管**：GitLab / GitHub

- **CI 工具**：GitHub Actions / GitLab CI

- **容器化与部署**：Docker + Docker Compose

- **生产环境**：Nginx + Gunicorn + Django + MySQL

- **测试环境**：自动化部署至测试服务器（Staging）

### 4.2 流程说明

`开发者提交代码 → 自动触发 CI 流水线    ├─ 代码检查 (lint / type check)    ├─ 单元测试 (pytest)    ├─ 构建 Docker 镜像    ├─ 部署至测试环境    └─ 手动触发 CD → 发布生产环境`

### 4.3 自动化任务

- 自动运行测试用例并生成测试报告；

- 自动生成API文档与构建前端静态文件；

- 自动推送版本镜像到镜像仓库（DockerHub / GitLab Registry）；

- 定期备份数据库与日志。
